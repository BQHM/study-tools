## 7. 重点准备的银行业务话术

### 开户业务怎么讲

开户业务不是简单插一条账户记录，它通常要经过客户信息校验、证件信息校验、产品参数校验、账户属性初始化、协议或介质绑定、核心交易落库和结果返回。开发时重点关注字段完整性、业务规则、重复开户防控、交易流水和异常返回码。

可追问：

- 如何防止重复开户？
- 开户失败后有没有补偿？
- 开户交易需要哪些校验？
- 开户表和客户表是什么关系？

### 存入业务怎么讲

存入业务核心是资金进入账户，要关注账户状态、交易金额、币种、产品规则、余额更新、交易流水和异常处理。开发时不能只看接口成功，还要关注交易是否可幂等、流水是否可追踪、失败时是否有明确错误码。

可追问：

- 存入交易如何保证幂等？
- 重复提交怎么处理？
- 余额更新和流水记录如何保证一致？
- 大额交易和普通交易规则是否不同？

### 支取业务怎么讲

支取业务比存入更敏感，因为涉及账户余额减少。核心校验包括账户状态、可用余额、冻结金额、支取规则、限额和交易密码或认证结果。技术上要重点讲事务边界、并发扣减、流水记录和异常回滚。

可追问：

- 如何避免余额扣成负数？
- 并发支取怎么控制？
- 支取成功但返回超时怎么办？
- 支取交易如何做冲正或补偿？

## 8. 八股优先级

面向这类银行 Java JD，八股优先级建议调整为：

1. MySQL / Oracle：索引、事务、锁、慢 SQL、执行计划。
2. Spring / Spring MVC / MyBatis：请求链路、事务失效、Mapper、动态 SQL。
3. Redis：缓存、分布式锁、幂等、防重复提交。
4. MQ：异步处理、可靠消息、重复消费、补偿。
5. Spring Cloud / Dubbo：注册发现、服务调用、超时、重试、熔断。
6. Linux：日志排查、常用命令、进程和端口。
7. Java 并发：线程池、ThreadLocal、锁、ConcurrentHashMap。
8. JVM：OOM、CPU 高、GC 基础排查。
