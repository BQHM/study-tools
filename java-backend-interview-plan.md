# Java 后端面试 6-8 周学习计划

> 来源参考：知识星球文章《Java 后端面试通关全阶指南（涵盖后端通用体系）》。
>
> 本计划不复制原文内容，而是按其核心优先级改造成个人执行版：项目与简历深挖 -> Java/MySQL/Redis -> 框架与系统设计 -> 计算机基础 -> 分布式高并发 -> JVM -> 面试冲刺。

## 0. 适用前提

你的当前情况：

- 有多个 Java 项目，但不少是 AI 辅助开发，需要重新接管和复盘。
- 加班导致项目细节遗忘，需要先恢复表达和基本盘。
- 短期目标不是包装成架构师，而是成为“能写代码、能讲项目、能做系统设计取舍”的 Java 后端候选人。
- 学习资料需要在公司电脑和家里电脑之间通过 Git 同步。

执行原则：

- 每天只推进一个主题。
- 每天必须留下产出，而不是只看文章。
- 所有答案沉淀到 `knowledge-answer-bank.md`。
- 所有学习进度记录到 `study-log.md`。
- 每次换电脑前更新 `HANDOFF.md` 并推送。

## 4. 第二阶段：Java + MySQL + Redis

周期：2-3 周。

重点不是刷题数量，而是把每个高频点变成“答案卡 + Demo + 项目关联”。

### Week 2：Java 核心与集合

主题顺序：

1. Java 基础：面向对象、equals/hashCode、String、异常、泛型。
2. HashMap：扩容、哈希冲突、红黑树、线程不安全。
3. ConcurrentHashMap：JDK 7/8 区别、CAS、synchronized、size。
4. ArrayList：扩容、fail-fast、和 LinkedList 对比。

每日产出：

- [ ] `knowledge-answer-bank.md` 新增或完善 2-3 张答案卡。
- [ ] `java-core-lab` 跑 1 个相关 Demo。
- [ ] `study-log.md` 记录今日卡壳点。

自测：

```text
随机抽 HashMap、ConcurrentHashMap、ArrayList 任意一题，
能画结构图并讲出 put/add 的关键流程。
```

### Week 3：并发与 MySQL

并发主题：

1. volatile。
2. synchronized / ReentrantLock。
3. ThreadLocal。
4. 线程池。
5. AQS。

MySQL 主题：

1. 索引和 B+Tree。
2. 事务 ACID。
3. 隔离级别和 MVCC。
4. undo log / redo log / binlog。
5. 慢 SQL 排查。

自测：

```text
能解释：
1. volatile 为什么不能保证 i++ 原子性。
2. Spring 事务和 MySQL 事务分别解决什么问题。
3. 一条慢 SQL 从哪里开始排查。
```

### Week 4：Redis

主题顺序：

1. Redis 为什么快。
2. 常见数据结构和场景。
3. 缓存穿透、击穿、雪崩。
4. 缓存一致性。
5. 分布式锁。
6. Redis Lua。

项目关联：

- FixLedger：提醒去重、JWT 黑名单、缓存钩子。
- 高并发项目：库存预热、Lua 扣减、防重复购买。

自测：

```text
能讲清：
1. Redis 缓存失败时核心业务怎么降级。
2. Redis Lua 为什么只能保证 Redis 内部原子，不能保证跨系统事务。
3. 缓存和数据库不一致怎么处理。
```

## 6. 第四阶段：计算机基础

周期：按目标公司安排，建议先 0.5-1 周补底。

计网必会：

- TCP 三次握手、四次挥手。

- HTTP/HTTPS。

- 死锁条件和避免。
- 用户态/内核态。
- IO 多路复用。

算法最低要求：

- 数组、链表、栈、队列。
- 哈希表。
- 二分。
- 双指针。
- 滑动窗口。
- 简单动态规划了解。

验收：

```text
常见计网/OS 题能讲机制；
```

## 7. 第五阶段：分布式与高并发

周期：1-2 周，取决于是否马上做高并发项目。

主题：

- 分布式 ID。
- 分布式锁。
- MQ 可靠性、重复消费、死信队列。
- 限流算法。
- 接口幂等。
- 缓存一致性。
- 分库分表基本思路。
- 服务降级、熔断、重试。

学习方式：

- 不先开大项目。
- 先把秒杀系统设计题讲熟。
- 再做一个最小可运行版本。

最小项目切片：

```text
Slice 1：活动发布 + MySQL 库存扣减
Slice 2：库存预热 + Redis Lua 扣减
Slice 3：MQ 异步下单 + 唯一索引防重复
Slice 4：超时取消 + 库存回滚
Slice 5：JMeter 压测报告
```

验收：

```text
能讲清每个切片解决什么问题；
能说出失败场景和补偿方式；
不在简历上写没亲手做过的能力。
```

## 10. AI Agent 工程师 JD 对齐路线

用户当前看到的 AI Agent 工程师 JD，要求的不只是 Java 八股，也不是纯算法训练，而是：

```text
Java 后端基本盘
+ LLM 应用工程
+ Agent 工具调用、记忆和规划
+ B 端产品落地和方案表达
```

因此后续学习需要做一个调整：

```text
每个八股知识点，都要绑定到 Agent 项目中的工程问题。
```

### 岗位能力拆解

| JD 要求 | 学习重点 | 项目落点 |
| --- | --- | --- |
| 需求拆解和项目排期 | 技术方案、任务拆分、风险说明 | `interview-assistant` 小切片计划 |
| 原型到上线 | 接口、状态、存储、日志、异常、降级 | 简历分析、面试评估、报告链路 |
| 多模态交互 | 文件、图片、语音输入的工程接入 | 简历文件解析，后续语音面试 |
| 工具调用 | Function Calling / Tool Calling | 出题工具、评分工具、知识库检索工具 |
| 记忆增强 | 会话记忆、长期记忆、用户画像 | 用户薄弱点记忆表 |
| 自主规划 | Plan-and-Execute、状态机 | 生成学习计划和下一轮训练任务 |
| 大模型生态 | Spring AI、RAG、Agent 框架 | AI 面试准备 Agent |

### 八股优先级调整

为了适配这个 JD，八股学习优先级调整为：

1. Spring Boot、REST 接口、参数校验、统一响应、异常处理。
2. MySQL 表设计、索引、事务、唯一约束。
3. Redis 缓存、限流、会话状态、分布式锁。
4. 线程池、异步任务、MQ。
5. HashMap、ConcurrentHashMap、ThreadLocal。
6. JVM 排查基础。

这不代表集合和并发不重要，而是面向 AI Agent 产品落地时，要优先能解释“系统怎么稳定运行、怎么存状态、怎么降级、怎么追踪任务”。

### interview-assistant 新定位

`interview-assistant` 后续可以从“智能面试助手 MVP”升级为：

```text
AI 面试准备 Agent
```

核心流程：

```text
上传简历
-> Agent 分析岗位匹配度
-> 生成学习计划
-> 调用知识库出题
-> 用户回答
-> Agent 评分
-> 更新薄弱点记忆
-> 生成下一轮训练计划
```

### 后续 4 周强化线

1. 第 1 周：LLM 接入、Prompt、结构化输出、失败兜底。
2. 第 2 周：Tool Calling，设计并实现 1 个后端工具调用闭环。
3. 第 3 周：RAG 和 Memory，把答案库和用户薄弱点接入训练流程。
4. 第 4 周：Agent 工作流，形成一页方案、一张流程图和 1 分钟项目话术。

详细计划见 `ai-agent-jd-study-plan.md`。

## 12. 本计划和原教程的差异

原教程是通用 Java 后端面试路径。本计划做了这些个人化调整：

- 把“项目与简历深挖”放到最重，因为你当前项目存在 AI 辅助和遗忘问题。
- 把 `interview-assistant` 设为主接管项目，而不是直接追求新大项目。
- 把高并发项目推迟到 Java/MySQL/Redis/Spring 之后，避免再次变成 AI 生成壳。
- 增加 AI 工程化学习线，包括 RAG、Prompt、结构化输出、幻觉处理。
- 根据 AI Agent 工程师 JD，增加 Tool Calling、Memory、Planning、Agent 工作流和技术方案表达。
- 增加跨电脑 Git + handoff 工作流，保证学习不会断档。
