## 3. 求职主线

### 主线：银行 Java / 金融科技 Java

定位：

```text
银行核心业务 Java 开发工程师，具备存款账户、开户、存入、支取等交易开发经验，当前参与数字人民币业务开发。
```

适合投递：

- 银行科技 Java 开发。
- 金融科技 Java 开发。
- 银行外包核心系统开发。
- 存款、账户、支付、数币、信贷相关 Java 岗。
- 要求 2-4 年或 3 年左右的 Java 岗。

优势：

- 有真实银行业务开发经历。
- 有长亮科技存款组经历。
- 做过开户、存入、支取这类银行交易。
- 有产品研发背景，并参与过多个银行客户 POC / 驻场支持。
- 当前有宁波银行数币驻场经历。
- 比纯学习项目候选人更有业务可信度。

短板：

- 工作年限约 2 年多，部分 3-5 年 JD 会略低。
- 微服务治理、Spring Cloud、Dubbo、Redis、MQ、ES、Jenkins、Sonar 需要补强。
- 高并发、高可用、海量业务架构不能包装成主导经验。
- 对公信贷产品设计和项目管理经验目前不足。

### 银行 Java 岗 1 分钟版本

我 2024 年毕业，2024 年 4 月入职长亮科技，主要在存款组做 Java 开发，到现在大概两年多。我的工作方向偏银行核心业务和存款产品研发，参与过长亮 V8.7 基板相关的开户、存入、支取/存取款等交易开发，也参与过青海农信、安徽农信、广农商、郑州农商等 POC，以及浦发银行新核心系统驻场设计和开发。2026 年 6 月开始，我在宁波银行驻场参与数币业务开发，进一步接触银行实际业务系统和交易流程。

### AI Agent 岗 30 秒版本

我现在的主业经历是 Java 后端和银行业务开发，同时在补 AI 应用工程化方向。我不做模型训练，重点关注怎么把大模型能力落到业务系统里，比如 Prompt 模板、结构化输出、工具调用、RAG、用户记忆、异步任务、降级和成本控制。我正在把 `interview-assistant` 改造成 AI 面试准备 Agent，用它练习从需求拆解、后端实现到项目表达的完整流程。
